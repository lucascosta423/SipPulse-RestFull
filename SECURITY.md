# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in SipPulse-RestFull, please report it responsibly.

**Please do not open a public issue for security vulnerabilities.**

Instead, please:

1. **Email** us at [security-email-placeholder] with details of the vulnerability
2. **Describe** the vulnerability and its potential impact
3. **Include** steps to reproduce, if possible

## Response Timeline

- We will acknowledge receipt of your report within **48 hours**
- We aim to send an initial assessment within **5 business days**
- We will work to resolve the issue and release a patch as soon as possible

## Scope

This policy applies to:

- The SipPulse-RestFull codebase
- Official Docker images published on Docker Hub
- GitHub Actions workflows

## Disclosure

We follow responsible disclosure. We kindly ask that you give us reasonable time to address the issue before public disclosure. We will credit you for the discovery unless you prefer to remain anonymous.
